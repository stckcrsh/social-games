# Functional Requirements

## Manager Features

Managers must be able to:

- interact with their wrestler
- submit weekly requests
- allocate influence money
- place bets

## Wrestler System

The system must support:

- persistent personalities
- emotional changes
- evolving relationships
- memory tracking
- trust toward manager

## Storyteller Engine

The system must:

- assemble weekly show cards
- resolve segments and matches
- generate narration
- update world state

## Betting System

Users should be able to bet on:

- match winners
- betrayals
- major story outcomes

## Persistence

The system must persist:

- wrestlers
- managers
- rivalries
- shows
- submissions
- betting history

## Non Functional Requirements

- deterministic core logic
- AI used only for narration/dialogue
- persistent continuity between weeks
- modular architecture
